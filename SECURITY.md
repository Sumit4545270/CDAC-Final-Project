# Security Policy

## Supported Versions

This project follows a rolling-release model focused on security and stability.

| Version | Supported |
|--------|-----------|
| Main / Latest | ✅ Yes |
| Older versions | ❌ No |

Only the latest version of the project is actively maintained and receives
security updates.

## Reporting a Vulnerability

Security issues and vulnerabilities are handled responsibly.

### How to Report
- Open a **GitHub Issue** with the label `security`, or
- Use **GitHub Private Vulnerability Reporting**

### Response Timeline
- Initial response within **48–72 hours**
- Reported issues will be reviewed and validated
- If accepted, fixes or mitigations will be applied in a future update

### Scope

This repository demonstrates a **DevSecOps-based Infrastructure Automation**
workflow using tools such as:
- Terraform (Infrastructure as Code)
- Docker & Kubernetes
- Trivy for container security scanning
- SonarQube for static code analysis
- Prometheus & Grafana for monitoring
- GitHub Dependabot and code scanning

Please avoid publicly disclosing security vulnerabilities until they have been
reviewed and addressed.

## Responsible Disclosure

This project is intended for educational and demonstration purposes.
All security reports are handled in good faith.
